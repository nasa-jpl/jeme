# Model Biographical Files

This directory contains biographical information and team member listings for each Earth science model in the JEME dashboard.

## Files

- **RAPID.md** - Routing Application for Parallel computatIon of Discharge
- **CARDAMOM.md** - Carbon Data Model Framework
- **CMS-Flux.md** - Carbon Monitoring System Flux
- **ECCO.md** - Estimating the Circulation and Climate of the Ocean
- **ISSM.md** - Ice Sheet System Model
- **MOMO-CHEM.md** - Multi-mOdel Multi-cOnstituent CHEMical data assimilation

## Structure

Each model bio file contains:

1. **Overview** - Brief introduction to the model
2. **Background** - Historical context and development
3. **Key Features** - Main capabilities and characteristics
4. **Scientific Applications** - Use cases and research areas
5. **Technical Details** - Model specifications
6. **Team Members** - Organized by role:
   - Principal Scientists
   - Research Staff
   - Technical Staff
   - Postdoctoral Researchers
   - Software Engineers
   - Graduate Students (if applicable)
   - Collaborators
7. **Key Publications** - Important scientific papers
8. **Resources** - Links to repositories, websites, documentation
9. **Contact Information** - How to reach the team

## Important Note

**The team member names in these files are PLACEHOLDER EXAMPLES.**

Please update each file with:
- Actual current JPL team member names
- Correct roles and titles
- Accurate contact information
- Current publications and resources

## How to Update

1. Contact each model team lead for accurate team rosters
2. Verify roles and titles with JPL HR or team leads
3. Update publication lists with latest papers
4. Ensure all links are current and functional
5. Review technical specifications for accuracy

## Usage in Dashboard

These markdown files can be:
- Rendered on dedicated model information pages
- Used to populate "About the Team" sections
- Referenced in documentation
- Linked from the dashboard navigation

## Maintenance

- **Review Frequency**: Quarterly or when team changes occur
- **Responsibility**: Each model team should maintain their respective file
- **Version Control**: All changes should be committed to git with descriptive messages

## Questions?

For questions about this documentation structure, contact the JEME dashboard development team.
